# ACYS Прогноз — модель, коэффициенты и вывод полей

Документ описывает, как считается прогноз: что за поле, откуда берётся и с каким коэффициентом. Данные
приходят из двух источников — справочник бортов/оценки (на каждый (борт, месяц): оператор, тип, рыночная
стоимость, кресла, лизинг, дата поставки) и история рейсов (реальные рейсы по бортам).

Два `Data Type`:
- **Actuals** — по одной строке на реальный рейс, история → **вчера**.
- **Forecast** — прогнозные рейсы, **сегодня → дата запроса + `FORECAST_HORIZON_YEARS`**.

---

## 1. Коэффициенты (все настройки)

| Константа | Значение | Смысл / зачем |
|-----------|----------|---------------|
| `HISTORY_START` | 2022-07-01 | Самый ранний рассматриваемый месяц (нижняя граница CY2022). |
| `FORECAST_HORIZON_YEARS` | 2 | Прогноз доходит до CY(`as_of.year + 2 − 1`); для запроса 2026 → **CY2027** (последний полный CY). |
| `LEVEL_L` | 3 | Текущий **уровень** = медиана последних 3 десезонализированных месяцев (≤ frontier). Короткое окно = следует за текущей активностью. |
| `SEAS_K` | 6.0 | **Сжатие** сезонного коэффициента к 1.0 по числу наблюдений: `seasonal[c] = (Σratio + K)/(n + K)`. Редкие месяцы тянутся к «нет сезонности». |
| `FRONTIER_FRAC` | 0.6 | Месяц «полный», если рейсов ≥ 0.6 × медианы окна (иначе он ещё догружается/отстаёт и в фит не идёт). |
| `FRONTIER_WINDOW` | 9 | Сколько последних месяцев берётся для медианы полноты. |
| `FORECAST_PAX_LOAD_FACTOR` | 0.8 | `Total PAX = Total Seats × 0.8` (предполагаемая загрузка). |

В прогнозе **нет ни потолка роста, ни правила списания по рейсам** — размер флота берётся прямо из
справочника (§3), поэтому рост и списания — ровно те, что в справочнике.

---

## 2. Контрактный год (день-в-день)

**12-месячное окно, заканчивающееся В якорный день**, помечается годом начала. Для запроса `10-июл-2026`:

```
CY2025 = (10-июл-2025 , 10-июл-2026]   т.е. 11-июл-2025 … 10-июл-2026
CY2026 = (10-июл-2026 , 10-июл-2027]
```

Правило для даты рейса `d` относительно якоря `(am, ad)`: `CY = год(d) − 1`, если `(месяц, день) ≤ (am, ad)`,
иначе `год(d)`. То есть якорный день — **последний** день своего CY (`10-июл-2026 → CY2025`; `11-июл → CY2026`).
И факт, и прогноз следуют этому по каждому рейсу, поэтому якорный месяц делится между двумя CY (Jul→Jul).

---

## 3. Модель прогноза — флот летает непрерывно

**Принцип:** каждый борт из **последней ревизии справочника** летает **каждый** прогнозный месяц; никто не
списывается, если справочник его не убрал; будущие поставки входят в тот месяц, на который приходится дата
поставки. Объём на борт — из типичной активности типа; маршруты — из типичной сети типа у **этой** АК
(никогда маршрут, которым она не летала).

### 3.1 Фит (на суб-флот = `Aircraft Sub Series`), по месяцам ≤ frontier
- **`seasonal[1..12]`** — коэффициент месяца года. Для каждого календарного месяца `c`:
  `ratio_i = flights_i / mean(flights)`, затем `seasonal[c] = (Σ ratio + SEAS_K) / (n_c + SEAS_K)` (сжатие к 1.0).
- **`level`** — медиана последних `LEVEL_L` (=3) **десезонализированных** месячных сумм рейсов (`flights / seasonal[cal]`).
- **`base_fleet`** — медиана числа летавших бортов за последние `LEVEL_L` месяцев (тот флот, что дал `level`).

### 3.2 Frontier
`frontier` = самый свежий месяц с рейсами ≥ `FRONTIER_FRAC × median(последние FRONTIER_WINDOW месяцев)`. В фит
идут только месяцы ≤ frontier (отстающие свежие месяцы игнорируются для фита, но **остаются в actuals**).

### 3.3 На прогнозный месяц `m`, на суб-флот
- **`k` (рейсов на борт за месяц)** = `round(level / base_fleet × seasonal[cal(m)] × prorate)`.
  - `level / base_fleet` = десезонализированные рейсы **на один борт**.
  - `prorate` = покрытые-дни / дней-в-месяце = 1.0 для полных месяцев; **текущий** месяц покрывает сегодня → конец месяца (факт — более ранние дни), **последний** месяц покрывает начало → день `as_of + HORIZON`. Масштабирует и `k`, и распределение по дням одинаково.
- **`активный флот`** = борта суб-флота из последней ревизии с `Delivery Date ≤ конец m` (или null). **Никто не списывается.**
- **`пул маршрутов`** = рейсы **шаблонного месяца** суб-флота (последний тот же календарный месяц ≤ frontier; иначе последний месяц суб-флота). Это обычная сеть этого типа у оператора.
- **Генерация:** каждый активный борт делает `k` рейсов, циклически перебирая пул: `route = pool[((g−1) mod размер_пула) + 1]`, `g = 1..k`. Итого рейсов суб-флота = `k × |активный флот|` → растёт с поставками.

### 3.4 Вывод полей прогнозного рейса
| Поле | Источник |
|------|----------|
| Registration, Manufacturer, Aircraft Sub Series, Primary Usage, **Total Seats**, Delivery Date, Lease Type, Lease Dry/Wet, Operational Lessor | **назначенный борт** (последняя ревизия справочника) |
| Master Series, Operator | суб-флот / оператор запроса |
| IATA/ICAO Origin & Destination, Circle Distance, Actual Distance FR, Flight Time, Flight Time FR | **маршрут** из пула (реальный исторический маршрут типа) |
| Date | **распределяется по дням месяца** (текущий месяц: с сегодня; последний месяц: до дня `as_of + HORIZON`) — чтобы день-в-день контрактный год делил якорный месяц так же, как факт (окно Jul→Jul, а не Aug→Jul) |
| Contract Year | §2 к Date |
| **Total PAX** | `Total Seats × FORECAST_PAX_LOAD_FACTOR (0.8)` |
| Time Departed / Time Landed | null (у месячного прогноза нет времени суток) |
| **Agreed Value** | см. §4 |

## 4. Agreed Value

База на (борт, месяц) = **Indicative Market Value (US$m)** из справочника. Правила:

1. **Wet-лизинг → 0** (`Lease Dry/Wet = 'Wet'`).
2. **Факт, пропущенный месяц** → перенос последнего известного значения **вперёд** (заполняет пробелы справочника); ведущие пропуски берут самое раннее известное.
3. **Прогноз** → проекция **собственной амортизации** борта вперёд: `value(m) = last_actual_value + slope × месяцев_после`, `slope = LEAST(0, regr_slope(value, месяц))` (**зажат ≤ 0 — не растёт**), пол 0. Новый борт без истории держит справочное значение плоско.

### Средневзвешенное (`acys_summary_grouped`), на (борт, контрактный год)
| Колонка | Формула |
|---------|---------|
| Agreed Value on Inception | значение **первого** месяца CY |
| Agreed Value at End of Contract | значение **последнего** месяца CY |
| **Weighted Average Agreed Value** | **`(inception + end) / 2`** — среднее по времени прямой линии начало→конец, **всегда между ними** (устойчиво к внутригодовым скачкам рынка). |
| Activity-Weighted Average | `Σ(value × flights) / Σ(flights)` за CY (взвешено по рейсам). |

Wet-месяцы и неположительные значения исключаются из всех четырёх.

---

## 5. Вывод полей факта (для полноты)
Одна строка на реальный рейс. Атрибуты борта из справочника (достоверный оператор на (борт, месяц) = свежайшая
ревизия; wet-lease/ACMI-дубли схлопнуты). Маршрут/гео из источника рейсов. `Circle Distance` = большой круг
(haversine, км) между вылетом и прилётом; `Age = (Date − Delivery Date)/365.25`. Рейсы без кода аэропорта
вылета **или** прилёта отбрасываются. Нижняя граница по **дате**: `first_seen::date > (якорный день 2022)` —
рейс в якорный день при любом времени суток всё равно падает в CY2021 и отбрасывается, поэтому **CY2021 не появляется**.

## 6. `forecast.acys_forecast_coefficients` — коэффициенты для графиков

Одна строка на **(Operator, Master Series, Forecast Month)**, пишется моделью прогноза (обновление по
оператору: удаляет свои строки оператора, затем вставляет заново). Содержит все коэффициенты, стоящие за прогнозом.

| Колонка | Смысл |
|---------|-------|
| Operator | оператор |
| Master Series | широкий тип (для группировки на графиках) |
| Aircraft Sub Series | **ключ суб-флота**, по которому строится прогноз (фит / флот / маршруты) |
| Forecast Month / Calendar Month | 1-е число прогнозного месяца / месяц года (1..12, для сезонной кривой) |
| Frontier | последний полный факт-месяц (граница фита) |
| Level | десезонализированный текущий уровень рейсов (суб-флот) |
| Base Fleet | типичное число летавших бортов |
| Per Aircraft Rate | `Level / Base Fleet` = десезонализированные рейсы на борт |
| Seasonal Factor | `seasonal[Calendar Month]` |
| Proration | доля покрытых дней (1.0 кроме текущего и последнего месяцев) |
| Active Fleet | бортов летает в этом месяце (поставлены ≤ месяц) |
| Flights Per Aircraft | `k = round(Per Aircraft Rate × Seasonal Factor × Proration)` |
| Forecast Flights | `k × Active Fleet` — объём прогноза за месяц |
| Template Month | месяц-шаблон маршрутов |

Рецепты графиков:
- **Объём прогноза по времени:** `SELECT "Forecast Month", sum("Forecast Flights") … GROUP BY 1`.
- **Рост флота (приход поставок):** `sum("Active Fleet")` по `"Forecast Month"`.
- **Сезонная кривая:** `"Seasonal Factor"` по `"Calendar Month"` (distinct по `"Master Series"`).
- **Rate / level / base fleet:** одно значение на суб-флот.
- `Forecast Flights = Flights Per Aircraft × Active Fleet`, `Flights Per Aircraft = round(Per Aircraft Rate × Seasonal Factor × Proration)` — весь расчёт восстанавливается из строки.
